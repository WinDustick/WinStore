# Реестр документации базы данных WinStore (Oracle Edition)

**Версия реестра:** 2.2
**Статус:** Актуальный
**Область действия:** Ветка `oracle/`

Данный документ является единой точкой входа для навигации по технической документации базы данных. Документы сгруппированы по функциональным областям: от архитектуры и стандартов разработки до эксплуатации и спецификаций API.

---

## 1. Архитектура и Общие сведения

Документы этого раздела описывают концептуальное устройство базы данных, принятые архитектурные решения и ограничения платформы.

| Файл | Описание и Назначение |
| :--- | :--- |
| **`oracle/docs/README.md`** | **Главный файл ветки.** Содержит обзор миграции с MS SQL, системные требования (Oracle 21c XE), инструкцию по развертыванию и список известных ограничений (включая статус Unified Auditing в XE). |
| **`oracle/docs/database_documentation_oracle.md`** | **Архитектурный обзор.** Описание схемы данных, ER-диаграмма, принципы реализации EAV (Entity-Attribute-Value) для товаров, типы данных и глобальные настройки. |
| **`oracle/docs/adrs/ORCL-ADR-001_Initial_Branch_Setup.md`** | **ADR (Architecture Decision Record).** Обоснование создания отдельной ветки для Oracle, стратегия поддержки кода и изоляции специфичных для СУБД функций. |

---

## 2. Стандарты разработки

Нормативная база, регулирующая стиль написания кода и проектирование объектов БД.

| Файл | Описание и Назначение |
| :--- | :--- |
| **`oracle/docs/SQL_style_guide.md`** | **Кодекс стиля (Style Guide).** Правила именования объектов (Snake Case), форматирование PL/SQL кода, требования к комментариям и структуре файлов. |
| **`docs/aic_guide/database.md`** | **Философия БД.** Общие принципы "Absurdly Ideal Code" применительно к базам данных: "Dumb Database", запрет на бизнес-логику в триггерах, идемпотентность скриптов. |

---

## 3. Спецификации серверного API (PL/SQL Packages)

Детальные технические спецификации пакетов, реализующих слой доступа к данным (DAL) и бизнес-логику на стороне сервера.

| Файл | Модуль | Описание |
| :--- | :--- | :--- |
| **`oracle/docs/pkg_product_spec.md`** | **Catalog** | Спецификация пакета `pkg_product`. Описание процедур поиска, фильтрации, пагинации товаров и агрегации JSON-атрибутов. |
| **`oracle/docs/pkg_order_spec.md`** | **Orders** | Спецификация пакета `pkg_order`. Описание транзакционной модели создания заказа, конечного автомата статусов и работы с позициями. |
| **`oracle/docs/pkg_payment_spec.md`** | **Payments** | Спецификация пакета `pkg_payment`. Описание логики финансовых транзакций, интеграции с внешними шлюзами и валидации переходов. |
| **`oracle/docs/pkg_user_spec.md`** | **Identity** | Спецификация пакета `pkg_user`. Описание процедур аутентификации, регистрации, безопасности паролей и поиска пользователей. |
| **`oracle/docs/rgr_analytics_spec.md`** | **Analytics** | Спецификация аналитического модуля `pkg_rgr_analytics`. Описание OLAP-запросов, сводных отчетов и сегментации клиентов (RFM). |

## 4. Внутренняя логика и Автоматизация

Документация по объектам, работающим автоматически (триггеры, джобы) и скрытым от прямого вызова.

| Файл | Описание и Назначение |
| :--- | :--- |
| **`oracle/docs/trigger_registry.md`** | **Реестр триггеров.** Полный список всех триггеров в системе с классификацией по назначению: генерация ключей, аудит, временные метки. Описание логики срабатывания. |

---

## 5. Эксплуатация и Безопасность (Ops & Sec)

Руководства для администраторов базы данных (DBA) и специалистов по информационной безопасности.

| Файл | Описание и Назначение |
| :--- | :--- |
| **`oracle/docs/security_operations.md`** | **Руководство по безопасности.** Регламент аудита, интерпретация логов `BusinessAuditLog`, матрица ролевого доступа (RBAC), политика хранения логов. |
| **`oracle/docs/recovery.md`** | **Disaster Recovery.** Инструкции по резервному копированию (Data Pump Export/Import), сценарии восстановления после сбоев и стратегия бэкапов. |
| **`oracle/docs/tde_setup_guide.md`** | **Настройка TDE.** Пошаговое руководство по первичной настройке Transparent Data Encryption (TDE) в среде Oracle 21c XE (Docker). |
| **`oracle/docs/tde_implementation.md`** | **Внедрение TDE.** Практические аспекты внедрения TDE, включая устранение неполадок с кошельком (Keystore) и проверку статуса шифрования. |
| **`oracle/docs/standalone_rgr_spec.md`** | **Дополнительные материалы.** Описание вспомогательных скриптов и академических примеров, не вошедших в основной контур. |

## 6. Инструментарий и Тестирование

Документация по вспомогательным инструментам генерации данных и тестирования.

| Файл | Описание и Назначение |
| :--- | :--- |
| **`oracle/docs/data_generation.md`** | **Data Seeding Manual.** Технический регламент подсистемы генерации тестовых данных. Архитектура скриптов, управление кэшем ID и сценарии полного сброса. |
| **`oracle/docs/data_seeding_rerun.md`** | **Итеративная генерация данных.** Описание принципов и сценариев повторного запуска скриптов для "долива" данных без полного сброса БД. |

### Порядок изучения для нового разработчика

1.  **Start:** `oracle/docs/README.md` — Развернуть локальную базу.
2.  **Standards:** `oracle/docs/SQL_style_guide.md` — Понять, как писать код.
3.  **Data:** `oracle/docs/data_generation.md` — Наполнить базу данными.
4.  **Logic:** Изучить спецификации пакетов (`*_spec.md`) по необходимому модулю.
5.  **Ops:** `oracle/docs/security_operations.md` — Понять, как работает аудит и права доступа.
